# 2026-06-14

## 高要求代码审查 — 快捷翻译浏览器插件

审查了全部 13 个源文件，发现问题如下：

### 严重（3）
1. `config.js`：百度/有道/腾讯 API 密钥明文硬编码，CRX 分发后任何人可读取
2. `engine-tx.js + background.js`：腾讯 SecretKey 每次随 sendMessage 明文传输
3. `content-debug.js:113`：`dbgList.innerHTML` 拼接了来自页面的文本，XSS 风险

### 警告（8）
1. `doBlocks()` 翻译失败无重试上限，所有引擎失败时死循环
2. `cancelled` 模块级变量存在竞态，多次快速触发 toggle 会导致旧任务继续执行
3. 百度翻译只取 `trans_result[0].dst`，长文本分段结果被截断
4. MyMemory `match >= 0.9` 阈值过严，大量正常翻译被静默丢弃
5. `chrome.storage.local` 缓存无 TTL / 大小限制，长期会超 10MB 上限
6. `pickEngine` 轮询调度存在尾部引擎饥饿
7. `doBlocks` 等 pickEngine 时无超时退出机制，限速后卡死
8. 有道 `truncate()` 字符数/字节数问题（待验证是否影响签名）

### 建议（5）
1. MD5 实现内联在 engine-bd.js，应抽取为 lib-md5.js
2. `hashKey` 自定义双哈希有碰撞风险，可直接用原文截断作 key
3. 语言对（en→zh）分散硬编码多处，应统一常量
4. manifest.json 混用 Firefox gecko 字段与 chrome.* API，跨浏览器策略不清晰
5. dbgCount.innerHTML 建议改为 DOM API 构建
